# Windsurf Memories, Rules, AGENTS.md, and Activation Modes

Source: https://docs.windsurf.com/windsurf/cascade/memories
Last updated: 2026-04-27 (Wave 14+, Windsurf 2.0 era)

Persist context across Cascade conversations with auto-generated memories and user-defined rules at global, workspace, and system levels.

## Choosing the Right Feature

| Feature | What it does | How activated | When to use |
|---|---|---|---|
| **Rules** | Tell Cascade *how to behave* (e.g. "use bun, not npm") | `always_on`, `glob`, `model_decision`, or `manual` | Coding conventions, style guides, project constraints |
| **AGENTS.md** | Location-scoped rules with zero config | Automatic — root=always-on, subdir=glob | Directory-specific conventions without frontmatter |
| **Workflows** | Prompt templates for repeatable multi-step tasks | Manual only via `/[workflow-name]` slash command | Deployments, PR reviews, release checklists |
| **Skills** | Multi-step procedures bundled with supporting files | Dynamically invoked by model, or `@mention` | Complex tasks needing reference files |
| **Memories** | Context Cascade auto-generates during conversations | Automatic retrieval when relevant | One-off facts; prefer Rules for durable knowledge |

**Recommendation**: For knowledge you want Cascade to reliably reuse, write it as a Rule or add it to `AGENTS.md` rather than relying on auto-generated Memories. Rules are version-controlled, shareable, and give explicit control over activation.

## Memories

- Auto-generated by Cascade during conversation when it encounters useful context
- You can also prompt: "create a memory of ..."
- Stored locally in `~/.codeium/windsurf/memories/`
- Associated with the workspace they were created in — NOT shared across workspaces
- NOT committed to your repository
- Creating and using auto-generated memories do NOT consume credits

## Rules

### Scopes and Storage Locations

| Scope | Location | Notes |
|---|---|---|
| Global | `~/.codeium/windsurf/memories/global_rules.md` | Single file, always on. Limited to 6,000 characters. |
| Workspace | `.windsurf/rules/*.md` | One file per rule, each with its own activation mode. Limited to 12,000 characters per file. |
| AGENTS.md | Any directory in your workspace | Processed by same Rules engine. Root=always-on, subdir=auto-glob. |
| System (Enterprise) | OS-specific (e.g. `/etc/windsurf/rules/`) | Deployed by IT, read-only for end users. |

### Activation Modes (trigger field in frontmatter)

| Mode | trigger: value | How it reaches Cascade | Context cost |
|---|---|---|---|
| **Always On** | `always_on` | Full rule content in system prompt on every message | Every message |
| **Model Decision** | `model_decision` | Only description shown in system prompt; full content read on demand | Description always; full on demand |
| **Glob** | `glob` | Applied when Cascade reads/edits a file matching `globs` pattern | Only when matching files touched |
| **Manual** | `manual` | NOT in system prompt; activated by typing `@rule-name` | Only when @mentioned |

The global rules file and root-level AGENTS.md files don't use frontmatter — they are always on.

### Example workspace rule with frontmatter

```markdown
---
trigger: glob
globs: **/*.test.ts
---

All test files must use `describe`/`it` blocks and mock external API calls.
```

### Rules Discovery
- Current workspace and sub-directories: all `.windsurf/rules` directories
- Git repository structure: searches up to git root
- Multiple workspace support: deduplicated, shortest relative path shown

### Best Practices
- Keep rules simple, concise, and specific
- No need for generic rules ("write good code") — already baked into Cascade training
- Format using bullet points, numbered lists, and markdown
- XML tags can group similar rules together

## AGENTS.md

`AGENTS.md` (or `agents.md`) is processed by the same Rules engine as `.windsurf/rules/` but with activation inferred from file location — no frontmatter needed.

- **Root directory**: Always-on rule — full content in system prompt on every message
- **Subdirectories**: Glob rule with auto-generated pattern `<directory>/**` — applied only when Cascade reads/edits files inside that directory

### Example structure

```
my-project/
├── AGENTS.md                    # Global instructions for entire project
├── frontend/
│   └── AGENTS.md                # Instructions specific to frontend code
├── backend/
│   └── AGENTS.md                # Instructions specific to backend code
└── docs/
    └── AGENTS.md                # Instructions for documentation
```

### AGENTS.md vs Rules

- AGENTS.md: no frontmatter, zero config, location-based scoping
- Rules (`.windsurf/rules/`): explicit activation mode, version-controlled, shareable with team
- Both processed by same engine
