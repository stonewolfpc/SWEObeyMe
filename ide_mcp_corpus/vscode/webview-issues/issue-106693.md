# GitHub Issue #106693: Webview refreshes to previous state after clicking another open editor

**Issue URL:** https://github.com/microsoft/vscode/issues/106693

**Reporter:** nsharma1989  
**Date:** Sep 14, 2020  
**VSCode Version:** 1.49  
**OS:** Darwin x64 19.5.0

## Description

### Steps to Reproduce
1. Open a react webview perform state changes.
2. Open settings preferences
3. Click back to the editor where webview is opened.

### Result
The webview refreshes back to the very first state of launching webview.

### Important Note
The extension has the flag `retainContextWhenHidden : true`. This problem wasn't there in previous versions.

### Does this issue occur when all extensions are disabled?
Yes

## Analysis

This issue describes **exactly the problem** occurring in SWEObeyMe cockpit:
- Webview has `retainContextWhenHidden: true`
- When Settings UI is opened, webview becomes hidden
- When returning to webview, it refreshes to initial state
- The "long list of 152 options" that appears is likely the initial state or a fallback state

## Root Cause

VSCode bug where `retainContextWhenHidden` does not properly preserve webview state when:
1. Webview becomes hidden (e.g., Settings UI opens)
2. Webview regains visibility
3. VSCode incorrectly resets the webview to its initial HTML instead of preserving the current state

## Potential Solutions

1. **Disable retainContextWhenHidden**: Use proper state persistence instead (we already have state-persistence.js)
2. **Monitor visibility changes**: Listen to `onDidChangeViewState` and restore state manually
3. **Force re-render on visibility change**: Detect when webview regains visibility and re-apply current mode/tab state
