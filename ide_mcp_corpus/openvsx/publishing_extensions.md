# Publishing Extensions

This guide describes publishing to the public registry at open-vsx.org, which is managed by the [Eclipse Foundation](https://www.eclipse.org/).
Similar steps may apply to registries that are hosted elsewhere, but make sure to pass the correct URL to the [ovsx](https://www.npmjs.com/package/ovsx) tool.

```
ovsx
```

If you are the author of the extension you wish to publish, proceed with the steps below. If you are not the author, we suggest you first reach out to the author with an issue in their GitHub repo to request that they publish their extension to open-vsx.org. We've drafted a [template](https://github.com/open-vsx/publish-extensions/blob/master/docs/external_contribution_request.md) with suggested content for the issue.
If the extension is open source or otherwise permits redistribution, you can request that it be auto-published as it is updated at the [Visual Studio Marketplace](https://marketplace.visualstudio.com/).
