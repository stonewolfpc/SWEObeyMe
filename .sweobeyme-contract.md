## Rules (Hardcoded)
- All edits must use MCP tools (no direct file access)
- 700-line maximum (hard ceiling), 600-line warning (soft limit)
- Split files at 600 lines using refactor_move_block or extract_to_new_file
- Document everything for non-coders (10% minimum comment ratio)
- Zero warnings tolerated
- Zero console.log statements (auto-replaced with comments)
- Zero todo: comments without tracking
- Zero debugger or eval() statements
- Every write_file creates mandatory backup first
- Every tool call logged to stderr
- Use surgical workflows for complex multi-file operations
